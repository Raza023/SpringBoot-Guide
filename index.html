Spring vs Spring Boot

Understanding the evolution from configuration-heavy frameworks to production-ready magic.

Spring Framework

  • Widely used EE framework. EE: "Also called Jakarta EE(Enterprise Edition or previously named J2EE) is indeed an umbrella or abstract specification (expressed in a formal language). Anybody is open to develop and provide a working implementation of the specification. Enterprise JavaBeans(EJB) is part of the Jakarta EE specification. It's in the Jakarta EE API."
  • Simplifies Jakarta EE development
  • Helps developers to be more productive at work.
  • Focuses on several areas of an application.
  • Provides a wide range of features:
    • Spring core,
    • MVC,
    • AOP for cross-cutting logic "concerns that are spread across various parts of your application like u have a lot of modules in your application having different concerns one is doing one thing, other one is doing completely a separate thing and they are communicating with each other" e.g., @AuditTrail,
    • Spring ORM AND Spring JDBC (for persistence layer)
  • Dependency Injection (helps to create loosely coupled applications.)

Coupling

Inter-Module Dependency

Dependency of one module on other modules. Coupling describes how much one module/class depends on another. Low coupling means a module has fewer relationships.

Tight Coupling

If two pieces of code are tightly coupled, they are strongly dependent on each other.

Example of High coupling: mail system is being used in employee system, client system, and server system.

Loose Coupling

If they are loosely coupled, they work mostly independently, only interacting when necessary.

Example: Modules interact through an interface (e.g., MessageService). Changing the mail provider doesn't require changes in the Employee or Client systems.

Cohesion

Module Integrity

Togetherness of module's elements. High cohesion indicates that functionalities within a module are strongly related.

Low Cohesion

A class or module tries to do too many unrelated things, creating maintenance challenges.

Example: Adder class containing both logic (add()) and UI methods (input(), display()).

High Cohesion

A class or module does one thing and does it well.

Example: Specialized Window class (UI has input() and display() which are ui methods) separated from Calculator class (has Add(), Subtract(), Multiply() and Divide() which are the logic methods).
VS

Spring Boot

Spring Boot is an extension of the Spring framework. It eliminates the boilerplate configuration associated with Spring, helping you get production-ready applications up and running rapidly.

It provides:

  • 1

    Flexibility

  • 2

    Provides an easiest way to code with annotation configurations, default code, less code.

  • 3

    Less time to develop standalone applications with less configurations.

  • 4

    Auto-configuration is a significant feature.

Key Benefits

Dependency Resolution

No need to worry about versions and dependencies.
For spring boot we use spring-boot-starter-web and for persistence and hibernate we use spring-boot-starter-data-jpa and we don't need to identify the version like we used to do in Spring framework.

Minimum Configuration

Drastically reduces setup. Simply add properties to application.properties for JDBC, JPA, and other integrations.

Embedded Server

Includes internal Tomcat. Run applications as standalone Java processes without external server deployment. Tomcat, WebLogic servers are required to run applications for spring.

Bean Auto Scan

Automatically detects components and beans based on project structure, eliminating manual configuration needs. For spring boot we can add @ComponentScan(). For spring I have to specify the component scan.

Health Metrics

Actuator provides detailed monitoring, including thread dumps and memory occupation, for better observability. For spring we use customized framework or any 3rd party service.

Auto-configuration

A significant feature that automatically configures your application based on the dependencies present on your classpath.
